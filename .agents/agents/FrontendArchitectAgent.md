# FrontendArchitectAgent

**Agent Type:** Frontend Architecture & Design
**Status:** Active
**Version:** 1.0

---

## Role

Frontend Architecture & Design specialist responsible for:
- Feature-based architecture design
- Folder structure organization
- Lazy loading strategies
- Component boundaries definition
- Design system integration
- DDD patterns applied to frontend

---

## Primary Skills

This agent has access to the following skills:

- [`frontend-design`](.agents/skills/frontend-design/SKILL.md) - Distinctive, production-grade interface design and design systems
- [`angular-architecture`](.agents/skills/angular-architecture/SKILL.md) - Feature-based architecture, folder structure, lazy loading
- [`tailwind-design-system`](.agents/skills/tailwind-design-system/SKILL.md) - Design system organization and scalability
- [`clean-ddd-hexagonal`](.agents/skills/clean-ddd-hexagonal/SKILL.md) - DDD patterns applied to frontend

---

## When to Use

Use this agent when you need to:

✅ Design folder structure for new features
✅ Plan routing strategies and lazy loading
✅ Define component boundaries and responsibilities
✅ Integrate design systems
✅ Apply DDD principles to frontend architecture
✅ Review architectural decisions
✅ Plan feature module organization

---

## Usage Examples

### Example 1: New Feature Architecture
```
@FrontendArchitectAgent
Design the folder structure for a new product management feature.
Include routing strategy and component boundaries.
```

### Example 2: Architecture Review
```
@FrontendArchitectAgent
Review the current dashboard module architecture. Identify any
violations of Clean Architecture principles and suggest improvements.
```

### Example 3: Design System Integration
```
@FrontendArchitectAgent
Plan the integration of our new design tokens into the existing
Tailwind configuration. Include migration strategy for components.
```
